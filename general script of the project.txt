Home Nutrition — Project Summary
What It Is
A bilingual (Arabic/English) e-commerce website for a gym supplements store in Constantine, Algeria. Customers order online, the admin calls to confirm, then ships. No payment gateway — cash on delivery.

Tech Stack
Backend: Node.js v24 + Express.js + SQLite (node:sqlite built-in)
Frontend: Pure HTML/CSS/Vanilla JS — no framework, no bundler
Auth: JWT (8h), stored in localStorage
File uploads: multer → uploads/products/
Customer Features
Feature	How It Works
Bilingual UI	Arabic (RTL, default) / English (LTR) — toggle in top bar
Homepage	Hero banner + category filter cards + product grid
Live search	Filters product grid in real time (no API call)
Category filter	Click a category card to filter — shows item count
Product detail page	Flavor circles + weight buttons + qty control + stock label
Flavor selector	Colored circles — each flavor has independent weights, prices, stock
Weight selector	Pill buttons per flavor — out-of-stock weights are grayed out
Stock indicator	"✓ In Stock", "✓ In Stock — 3 left" (when ≤10), "✗ Out of Stock"
Qty cap	Cannot exceed available stock (HTML max + JS clamp + server validation)
Add To Cart	Adds selected variant × qty; respects stock already in cart
Buy Now	Same as Add To Cart but navigates directly to cart page
Shopping cart	Persists in localStorage; shows flavor · weight per item
Cart qty controls	+/− per item, capped at stock; stale items auto-purged
Checkout form	First name, last name, wilaya, commune, address (optional), email, phone
Home delivery	If address is filled → home delivery; if empty → store pickup
Order submission	Server validates and calculates total independently
Order confirmation screen	Shows order reference (e.g. ORD-1782527663360) after placing
Admin Features
Feature	How It Works
Secure login	Username + password → JWT valid 8 hours
Stats cards	Total products / In Stock / Out of Stock counts
Product table	All products with image, name, category, price, stock summary
Add product	Full form: names (AR+EN), category (AR+EN), price, description, image
Edit product	Pre-fills all fields including full variant tree
Delete product	FK-safe deletion: variants → flavors → order_items → product
Image upload	Upload JPEG/PNG/WebP (max 5MB) → stored in uploads/products/
Flavor system	Per-product flavors with hex color picker for the circle button
Weight variants	Per-flavor weights with independent price, stock, image
Stock tracking	NULL = unlimited (manual toggle); integer = tracked (auto-derives in_stock)
Orders table	Ref, customer, location, home delivery flag, items, total, status
Order status filter	Tabs: All / Pending / Confirmed / Cancelled — with live counts
Home delivery badge	"🏠 Home delivery: Yes" or "📦 Home delivery: No" per order row
Order details modal	Full customer info + delivery address + itemized list with flavor/weight
Confirm order	Decrements stock atomically → status = confirmed (idempotent)
Cancel order	Status = cancelled, no stock change
Delete single order	Only works on cancelled orders
Delete all cancelled	Bulk delete with count in confirmation dialog
Key Design Decisions
Stock only decreases on confirm, not on order placement — because the owner calls every customer before dispatching
Variant prices are independent — 300g Chocolate can cost differently than 1kg Natural
Flavor/weight labels are snapshotted in order_items — admin can always read what was ordered even if variants are later renamed/deleted
One admin account — single store owner model
No payment gateway — cash on delivery, manual process
